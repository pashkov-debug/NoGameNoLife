Фаза 2. Вертикальный срез “пустая локация”

Цель: проверить core loop.

Что делаем
экран списка локаций

чтение состояния локации из БД
если локация пустая и cooldown истёк:
lock на локацию
rule engine решает, надо ли создать событие
AI пишет текст описания
событие сохраняется
игрок видит событие
игрок отправляет действие текстом
AI парсит действие в intent JSON
rule engine применяет результат
AI формирует текст результата
лог пишется в БД
Результат

Игрок заходит в локацию, получает описание, делает действие, получает исход.
